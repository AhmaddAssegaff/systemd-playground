# Basic Systemd Commands
This document contains the most commonly used systemd commands while developing and managing Linux services.

---

## Reload Systemd Configuration
Reload systemd after creating or modifying a service file.
```bash
sudo systemctl daemon-reload
```

---

## Start Service
Start a service immediately.
```bash
sudo systemctl start hello-world
```

---

## Stop Service
Stop a running service.
```bash
sudo systemctl stop hello-world
```

---

## Restart Service
Restart a service.
```bash
sudo systemctl restart hello-world
```

---

## Reload Service
Reload service configuration without a full restart.
```bash
sudo systemctl reload hello-world
```

---

## Check Service Status
Display current service status, PID, logs, and recent events.
```bash
sudo systemctl status hello-world
```

---

## Enable Service On Boot
Start the service automatically when the system boots.
```bash
sudo systemctl enable hello-world
```

---

## Disable Service On Boot
Disable automatic startup.
```bash
sudo systemctl disable hello-world
```

---

## View Service Logs
Show logs generated by the service.
```bash
journalctl -u hello-world
```

---

## Follow Logs In Real Time
Equivalent to `tail -f` for systemd logs.
```bash
journalctl -u hello-world -f
```

---

## Show Recent Logs
Display the last 50 log entries.
```bash
journalctl -u hello-world -n 50
```

---

## Show Logs Since Boot
Display logs generated since the current system boot.
```bash
journalctl -u hello-world -b
```

---

## Show Unit Definition
Display the service file currently loaded by systemd.
```bash
systemctl cat hello-world
```
Useful for verifying that systemd is loading the expected service file.

---

## Verify Service File
Validate service file syntax.
```bash
sudo systemd-analyze verify \
  /etc/systemd/system/hello-world.service
```

---

## List Failed Services
Display all failed units.
```bash
systemctl --failed
```

---

## Show Unit Search Paths
Display directories scanned by systemd.
```bash
sudo systemd-analyze unit-paths
```

---

## Reload Systemd Manager
Rarely needed.
```bash
sudo systemctl daemon-reexec
```
This restarts the systemd manager process while keeping services running.

---

## Inspect Service Startup Errors
Useful when a service fails to start.
```bash
journalctl -xeu hello-world.service
```

Common errors:
* `203/EXEC` → executable not found or not executable
* `Permission denied` → permissions or SELinux issue
* `No such file or directory` → invalid path
* `Unit not found` → service file not loaded

---

## Service Lifecycle
Typical workflow during development:
```bash
go build -o bin/hello-world .

sudo systemctl daemon-reload

sudo systemctl restart hello-world

sudo systemctl status hello-world

journalctl -u hello-world -f
```
